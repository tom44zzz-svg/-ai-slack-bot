# アプリ UX フロー（Web アプリ版）

## 目的

フィード投稿作成時の**脳のリソースを最小化**する。
ネタを投入 → フォーマット選択 → 構成案が自動出力 → テキストを埋めるだけ、の状態を作る。

---

## 画面遷移

```
[1] ネタ入力
      │
      ▼
[2] フォーマット選択（軸で絞り込み）
      │
      ▼
[3] 図解の確認・選択
      │
      ▼
[4] 自動生成（Web 検索＋制作ルール反映）
      │
      ▼
[5] 出力確認 ＋ セルフチェック結果
      │
      ▼
[6] 編集・エクスポート
```

---

## [1] ネタ入力

**入力項目**：
- ネタ本文（自由記述 or スプレッドシート 1 行貼付）
- ターゲット（任意：『40 代共働き夫婦』など）
- 伝えたいゴール（任意：『保存してもらいたい』『行動してもらいたい』等）
- 参照 URL（任意）

---

## [2] フォーマット選択

### 方式 A：3 軸で絞り込み（推奨）

```
Q1. どう切り出しますか？（フック）
  ○ 問いかけ
  ○ 数字（〇選・TOP N）
  ○ 約束（〇〇する方法）
  ○ 失敗警告（やりがちな〇〇）
  ○ 常識反転（実は〇〇）
  ○ 体験（〇〇した話）
  ○ 宣言（〇〇とは？）

Q2. トーンは？
  ○ ポジティブ
  ○ 普通
  ○ ネガティブ

Q3. 中身の形は？
  ○ リスト
  ○ ステップ
  ○ 比較
  ○ 深掘り
  ○ ストーリー
  ○ Q&A
  ○ ケース分岐

→ 該当フォーマット 2〜3 個が提示される
→ カードをクリックで選択
```

### 方式 B：フォーマット名から直接選択

慣れたユーザー向け。20 フォーマットをカテゴリー別にグリッド表示。

### 補助：AI 推薦

ネタ本文から自動で適切なフォーマットを推薦（ユーザーは Accept/Reject）。

---

## [3] 図解の確認・選択

選択したフォーマットに応じて、**推奨図解**が視覚プレビューで表示される。

```
[現在のフォーマット：f_case_branch ケース分岐型]

推奨図解：
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ compare_3_   │  │ recommend_by │  │ cards_2x2    │
│   column     │  │ _case        │  │              │
│ [preview]    │  │ [preview]    │  │ [preview]    │
└──────────────┘  └──────────────┘  └──────────────┘
       ✓ 選択中       ✓ 選択中           選択可能

他の図解を使いたい場合 → 図解ギャラリーを開く
```

**図解ギャラリー**では 20 種類すべてを ASCII プレビュー（将来は画像）で一覧表示し、
カテゴリー（2値比較 / N値比較 / 階層 / フロー / 分布 / 強調）でフィルタ可能。

## [4] 自動生成

裏側の処理：

1. **Web 検索**（Google / 公式サイト）
   - ネタに関連する**国・企業情報**を収集（制作ルール 3）
   - 個人ブログ系ドメイン除外
2. **制作ルール反映**
   - 文体：ですます調・句読点あり（rule_01）
   - NG ワード除外（「セゾン」単独使用等、rule_08）
   - 統一表記に変換（rule_06）
3. **構成生成**
   - `formats.yaml` のレシピに従い、各スライドの zone に要素を配置
   - `templates.yaml` の制限字数内でテキストを生成
   - 写真の雰囲気指定（rule_13 準拠）
4. **タイトル 3 案**
   - フォーマットの `title_templates` に基づき 3 パターン生成

## [6] 編集・エクスポート

- 各スライドのテキストを編集
- タイトル案を選択 / 書き換え
- キャプション本文を編集（これは別フォーマットで生成）
- **エクスポート**：
  - Markdown
  - Google ドキュメント
  - Canva 貼り付け用テキスト
  - スプレッドシート反映

## 制約事項

- 最終スライドの CTA は固定（cta-patterns.yaml の default_combination）
- 表紙の背景写真は過去投稿と重複しないよう履歴チェック（rule_14）
- 『FP 監修』を含む投稿はキャプションに 3 URL を必ず付与（rule_10）
