# Canva フォーマット仕様書

## この仕様書の目的

ネタ（スプレッドシート）から Canva 投稿を生成するときに、
「どのフォーマットに落とすか」「各スライドにどんな文字数・ビジュアルを入れるか」を
**一意に決められる**リファレンスを提供する。

人間が手作業で投稿を作るときも、将来 bot が自動生成するときも、
同じ辞書（`data/formats.yaml`）を参照する。

## 使い方

1. ネタを見て、`フォーマット一覧` から最も近い型を選ぶ
2. 該当フォーマットの定義を `data/formats.yaml` または `docs/format-catalog.md` で確認
3. 各スライドの `text_limits` と `visuals` に沿って原稿を作成

## 文字数ルールの共通原則

_棚卸し中に共通パターンが見えたらここにまとめる。想定項目：_

- タイトル：視認性重視、最大文字数
- 本文：1 スライドあたりの情報量上限
- キャプション／注釈の扱い
